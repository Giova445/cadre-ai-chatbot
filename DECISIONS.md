# DECISIONS.md — build log

A required deliverable: for each notable module and choice, who authored it — **directly by the lead (the "Queen" driving Claude Code)** vs. **generated by a specialized subagent and integrated** — and *why* that split. This is an honest record of the Claude Code workflow, not a marketing gloss.

**Authorship legend**

- **Lead-authored** — written directly by the lead in the main Claude Code session. Chosen for anything safety-critical, compile-sensitive, or where owning the exact logic matters for the live defense.
- **Subagent-generated, lead-integrated** — produced by a delegated subagent against the frozen contracts in [SPEC.md](SPEC.md), then reviewed, fact-checked, and wired in by the lead.
- **Subagent-generated + lead-modified** — delegated, then materially edited by the lead.

The governing principle: **the retrieval math, the guardrail decision, and the streaming route are lead-authored** (they are the parts that must be defended line-by-line and cannot be allowed to drift); **content, tests, UI polish, and docs are delegated** and integrated against fixed interfaces.

## Key trade-off decisions

Each is expanded with cost/benefit in [TRADEOFFS.md](TRADEOFFS.md); here is the decision and the one-line why.

### DEC-1 — No vector database
**Decision:** in-memory brute-force cosine over a bundled `data/embeddings.json`; no pgvector, no local/hosted vector DB.
**Why:** at ~8–10 docs a linear scan is sub-millisecond and **exact** (100% recall). A vector index solves scanning *millions* of vectors — a problem this corpus does not have — and a local index conflicts with the serverless ephemeral filesystem. Adding one here is the "custom build when a simple thing works" the brief penalizes.

### DEC-2 — Offline deterministic lexical embedder as a keyless fallback
**Decision:** ship `lexicalEmbed` (signed FNV-1a feature-hashing → 512-dim, L2-normalized) as the default embedder; upgrade to real `text-embedding-3-small` only when `EMBEDDINGS_API_KEY` is present.
**Why:** the app must be runnable and gradeable with **zero credentials**, and the whole pipeline (retrieval, threshold, guardrails, evals) must exercise identically offline and online. Using the *same* embedder at build and query time keeps cosine scores comparable. Cost: it is lexical, not semantic — hence the lower, calibrated threshold. It is swappable for real embeddings with one env var + a re-embed.

### DEC-3 — Custom plain-text streaming protocol (not the AI SDK UI protocol)
**Decision:** `/api/chat` streams raw text and carries guardrail metadata in `x-cadre-*` headers, rather than emitting the AI SDK's UI-message data-stream protocol.
**Why:** the offline grounded-stub path and the online LLM-stream path must look identical to the client, and the deterministic refusal/escalation paths never touch the model. A plain-text stream + header metadata unifies all four cases behind one tiny client reader and avoids coupling the UI to a protocol that assumes a model is always in the loop.

### DEC-4 — Provider-agnostic seam
**Decision:** all provider specifics (chat model factory, real embeddings client, base-URL overrides) live only in `lib/llm.ts`, behind an OpenAI-compatible client.
**Why:** the exact provider/model is locked when a key arrives; swapping OpenAI ↔ OpenRouter ↔ another endpoint touches one file. This mirrors Cadre's own model-agnostic stance. Cost: it targets the lowest-common-denominator OpenAI-compatible API rather than provider-specific features — acceptable here.

### DEC-5 — Deterministic `decide()` shared by the route and the eval runner
**Decision:** guardrail routing (pricing → refuse, human → escalate, weak → escalate, else answer) is a single pure function used by both the live route and `pnpm eval`.
**Why:** it makes "what the bot decides" testable **without spending an LLM token**, keeps Layer 1 guardrails un-prompt-injectable (the model is never asked to police pricing), and guarantees the evals test the exact logic that runs in production. `Decision` now carries `{mode, reason, citations, topScore, coverage}`. Cost: regex intent detection is coarse; it is backstopped by the retrieval threshold, the coverage guard (DEC-7), and the Layer 2 grounding prompt.

### DEC-7 — Grounding-coverage guard (refuse when the context doesn't actually support the claim)
**Decision:** even when retrieval clears `RETRIEVAL_THRESHOLD`, `decide()` computes `coverage` — the fraction of the query's *distinctive* terms (via `tokenize`, minus a small ubiquitous stop-set) present in the retrieved chunk text — and refuses with `reason: "unsupported"` (copy in `responses.unsupportedRefusal()`) when it falls below `COVERAGE_MIN = 0.4`.
**Why:** cosine similarity alone cannot separate a plausible **fake** service from a real one when their scores are nearly identical. The hallucination-bait case ("Do you offer a 24/7 managed AI hosting plan?") scored ~0.279 versus ~0.281 for a genuine-service question — indistinguishable by threshold. The coverage guard catches it because the query's distinctive terms ("hosting", "24/7", "money-back guarantee") are absent from the KB, so it refuses rather than confirm. This is a deliberate, **embedder-agnostic** grounding technique that works with real embeddings too, and it is what makes `adv-hallucination` pass. Lead-authored (safety-critical). Cost: an over-narrow query could be refused; mitigated by the 0.4 floor and the ubiquitous-term exclusion.

### DEC-6 — Mode-aware single guardrail knob
**Decision:** one `RETRIEVAL_THRESHOLD`, defaulting to ~0.35 for real embeddings and ~0.08 for the lexical embedder, with an env override.
**Why:** one calibrated cutoff is tunable against the golden set and explainable in review; making the default mode-aware keeps the offline mode honest (the two embedders live on different score scales) without a second code path.

## Notes for the live defense

- Anything **lead-authored** above can be walked line-by-line: the cosine math, the `decide()` branches, the route's failure→escalation fallbacks, and the lexical embedder.
- Delegated work (KB, evals, tests, UI, docs) was integrated against the **frozen contracts in [SPEC.md](SPEC.md)**, which is why parallel work did not collide.
- Divergences from [plan.md](plan.md), all documented in [SPEC.md](SPEC.md): the `EmbeddingsFile` gained `builtAt` + `thresholdHint`; the plan's `topK` was built as pure `rankChunks` + a bound `retrieve`; `decide()` gained the grounding-coverage guard (DEC-7) and a `coverage` field; and `core-book` is a `grounded` case (the KB explains how to book) rather than an escalation.
